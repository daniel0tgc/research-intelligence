## 3. Universal Code Rules

### Never generate these patterns
```
❌ Hardcoded API keys, credentials, or connection strings — use os.environ or pydantic Settings
❌ Raw string queries in Neo4j — always use parameterized Cypher ($param syntax)
❌ Raw SQL in route handlers — use db/queries.py functions only
❌ Synchronous DB calls in async FastAPI routes — use asyncpg and async Neo4j driver
❌ bare except: clauses — catch specific exceptions only
❌ print() for logging — use the logging module or structlog
❌ Blocking Claude API calls in route handlers — wrap in asyncio or background task
❌ Direct Neo4j session management in route handlers — use the graph/client.py context manager
❌ Mutable default arguments in Python functions
❌ f-strings with user input inserted into Cypher or SQL queries
```

### Always do
```
✅ State plan in 3–5 bullets before writing any code for a new task
✅ Type hints on every function signature (input and return)
✅ Pydantic models for all FastAPI request bodies and response schemas
✅ Handle empty state, loading state, and error state for every data-dependent UI component
✅ Guard clauses and early returns before the happy path
✅ Use async/await consistently — never mix sync and async in the same call chain
✅ Keep files under 200 lines — split into modules when exceeded
✅ Check off the Done.md task checkbox immediately after completion
✅ Write phase summary to Done.md when all phase tasks are complete
✅ All Neo4j queries parameterized with $variable syntax
✅ All environment variables loaded from .env via pydantic BaseSettings
```

## 4. Stack-Specific Rules

### Backend (Python / FastAPI)
- All routes in `backend/api/routes/` — one file per domain (ingest.py, graph.py, papers.py, reports.py, agenda.py, concepts.py)
- All route handlers typed with Pydantic request/response models — no untyped dicts returned
- All Neo4j queries in `backend/graph/queries.py` — never inline Cypher in routes
- All PostgreSQL queries in `backend/db/queries.py` — never raw SQL in routes
- Background tasks via FastAPI BackgroundTasks or asyncio — never blocking the event loop
- All Claude API calls in `backend/agents/` — never called directly from routes
- WebSocket event emission in `backend/api/events.py` — never emit directly from routes

### Database (PostgreSQL + pgvector)
- Schema defined in `backend/db/schema.sql` — run via psql, never edit live DB directly
- All async DB access via asyncpg connection pool in `backend/db/client.py`
- pgvector index: `CREATE INDEX ON chunks USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100)`
- Never store raw embeddings as Python lists — always use numpy arrays converted at the boundary

### Graph DB (Neo4j)
- All Cypher in `backend/graph/queries.py` as named async functions
- Always use `AsyncGraphDatabase.driver()` — never sync driver in async context
- Every query uses `$param` parameterization — no string interpolation
- Constraints and indexes defined in Phase 1 setup script — never create ad-hoc
- Louvain community detection via `gds.louvain.stream()` — requires Neo4j GDS plugin (installed in Phase 1)

### MCP Server (FastMCP)
- All tools defined in `backend/mcp/server.py`
- Each tool is an async function decorated with `@mcp.tool()`
- Tool docstrings are the tool descriptions exposed to Claude — write them clearly
- Server runs on port 8001 — FastAPI runs on port 8000

### Frontend (React + TypeScript + Vite)
- All components in `frontend/src/components/` — one file per component
- 3D graph managed exclusively in `frontend/src/components/Graph3D.tsx`
- Global state in `frontend/src/store/` via Zustand — never prop-drill graph data
- All FastAPI calls through `frontend/src/lib/api.ts` — never fetch() inline in components
- WebSocket connection managed in `frontend/src/lib/socket.ts` — one singleton
- Tailwind for all styling — no inline style objects unless Three.js requires it
- Three.js / R3F: always dispose of geometries and materials on unmount
- Never import directly from 'three' if @react-three/drei provides the same primitive

### Embedding (Voyage AI)
- Model: `voyage-large-2` via Voyage AI REST API (`https://api.voyageai.com/v1/embeddings`)
- Embedding dimension: 1536
- All embedding calls in `backend/ingestion/embed.py` — never inline elsewhere
- Batch embed chunks in groups of 128 — never embed one at a time in a loop

### Claude API
- Model: `claude-sonnet-4-6` unless Context.md specifies otherwise
- All prompts defined as constants at the top of each agent file — never hardcoded inline
- Always use structured output (response format JSON) for entity/relation extraction
- Stream responses for the embedded chat endpoint

## 6. Debugging.md Protocol (Non-Negotiable)

After completing ALL tasks in a phase, before writing the phase summary:

1. Open Debugging.md and find the section for the current phase
2. Run every check command exactly as written using the Bash tool
3. For each check that passes: mark it `[x]` in Debugging.md
4. For each check that fails:
   - Output the exact error message
   - Output the "Why this breaks" explanation from Debugging.md
   - Fix the root cause
   - Re-run the check
   - Do not proceed to Done.md phase summary until all checks for that phase are `[x]`
5. If a check requires manual action (e.g., browser interaction), output `MANUAL CHECK REQUIRED` with exact steps, then wait

**Never mark a phase complete in Done.md until every Debugging.md check for that phase is checked off.**

If a check fails after 3 fix attempts, output BLOCKED with the check reference.
